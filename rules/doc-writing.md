# 문서 작성 규칙 — mimir-practice

## 기본 원칙
1. 모든 문서는 한국어로 작성하되, 기술 용어(Ingester, Distributor, Compactor, MetricsQL 등)는 영어 그대로 사용
2. 제목은 `# [주제] — Mimir [문서유형]` 형식으로 통일
3. 문서 상단에 작성일, 작성자, 검토자 메타데이터 포함
4. 코드 블록에는 반드시 언어 지정 (```promql, ```yaml, ```bash)
5. 멀티 테넌시 관련 문서는 테넌트 ID 예시 포함

## 문서 구조
- **개요**: 목적과 테넌트 범위 (3-5문장)
- **구성**: 컴포넌트 설정 및 제한
- **운영**: 핵심 명령어 및 체크리스트
- **트러블슈팅**: 증상-원인-해결 테이블
- **참고**: 관련 문서 링크

## MetricsQL 문서화 규칙
- PromQL과 호환되는 문법 우선 사용
- Mimir 확장 함수 사용 시 주석으로 설명
- 테넌트 컨텍스트(X-Scope-OrgID) 명시

## 금지 사항
- 개인 식별 정보 포함 금지
- 테넌트 자격증명 문서에 포함 금지
- S3 버킷/키 정보 문서에 하드코딩 금지
